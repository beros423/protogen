# ML Protogen System 🧬

머신러닝과 베이지안 최적화를 활용한 지능형 단백질 발현 실험 설계 시스템

## 개요

이 시스템은 기존의 `protogen.py`를 확장하여, 단순한 매핑 파일 생성을 넘어 **반복적 학습-최적화 사이클**을 통해 단백질 발현 실험을 지능적으로 설계하는 도구입니다.

### 핵심 개념
- **적응형 실험 설계**: 초기 데이터로 ML 모델을 학습한 후, 베이지안 최적화로 최적의 실험 조건을 탐색
- **반복적 개선**: 실험 → 결과 수집 → 모델 업데이트 → 새로운 실험 설계의 지속적 사이클
- **자동화된 워크플로**: ML 모델 선택부터 매핑 파일 생성까지 전 과정 자동화

## 🚀 빠른 시작

### 1. 설치 및 실행
```bash
cd ml_protogen_system
pip install -r requirements.txt
streamlit run app.py
```

### 2. 프로젝트 생성
1. 사이드바 상단에서 "새 프로젝트 생성..." 선택
2. 프로젝트 이름과 설명 입력  
3. "생성" 버튼 또는 "샘플 프로젝트로 시작" 버튼 클릭

### 3. 워크플로 진행
- 각 단계는 자동으로 저장됩니다
- 사이드바에서 언제든지 다른 단계로 이동 가능
- 테스트 데이터 생성 기능으로 실험 없이 시스템 체험 가능

## 📋 워크플로

### 1단계: 프로젝트 설정
- 프로젝트 기본 정보 입력 (이름, 설명)
- 최적화 목표 및 실험 조건 설정
- 예상 사이클 수 및 샘플 크기 정의

### 2단계: 초기 데이터 입력
- 기존 실험 데이터 업로드 (CSV/Excel)
- 수동 데이터 입력 지원
- 데이터 유효성 검증 및 통계 확인

### 3단계: 초기 ML 모델 학습
- 다중 ML 모델 자동 학습 (Random Forest, Gradient Boosting, Linear Regression)
- 교차 검증을 통한 모델 성능 평가
- 최적 모델 자동 선택 및 저장

### 4단계: 반복적 최적화 사이클
#### 사이클 A: 베이지안 최적화로 실험 설계
- 현재 모델 기반 베이지안 최적화 실행
- 탐색(Exploration) vs 활용(Exploitation) 균형 조절
- 새로운 실험 조건 제안 (5-20개)

#### 사이클 B: 매핑 파일 생성
- 실험 조건을 플레이트 레이아웃으로 자동 변환
- Well 위치 및 Sample ID 자동 할당
- CSV/Excel 형식으로 매핑 파일 다운로드

#### 사이클 C: 실험 결과 입력 및 모델 업데이트
- 실제 실험 수행 후 결과 입력
- **테스트 데이터 생성**: 실험 없이 가상 결과로 테스트 가능
- 새 데이터로 모델 재학습 및 성능 개선 확인
- 다음 사이클로 자동 진행 또는 프로젝트 완료

### 5단계: 프로젝트 완료
- 전체 사이클 성과 분석 및 시각화
- 사이클별 모델 성능 개선 그래프
- 모든 실험 데이터 및 결과 내보내기

## 🎯 주요 기능

### 자동 저장 시스템
- **단계별 자동 저장**: 각 단계 완료 시 자동으로 프로젝트 상태 저장
- **수동 저장**: 💾 버튼으로 언제든지 즉시 저장
- **상태 복원**: 브라우저 종료 후 재시작해도 이전 상태 완전 복원

### 통합 사이드바
- **프로젝트 선택**: 상단 선택박스에서 프로젝트 간 빠른 전환
- **진행 상황 추적**: 현재 단계와 완료된 단계를 시각적으로 표시
- **단계 간 이동**: 완료된 단계로 자유롭게 이동 및 수정 가능

### 테스트 데이터 생성
- **가상 실험 결과**: 실제 실험 없이 시스템 동작 체험
- **노이즈 조절**: 실험 결과의 변동성 조절로 현실적인 시뮬레이션
- **성능 트렌드**: 개선/유지/악화 트렌드를 통한 다양한 시나리오 테스트

### 지능형 최적화
- **베이지안 최적화**: Gaussian Process 기반 효율적인 실험 공간 탐색
- **다중 모델 앙상블**: 여러 ML 모델의 성능을 비교하여 최적 모델 자동 선택
- **적응적 샘플링**: 모델 불확실성이 높은 영역을 우선적으로 탐색

## 📊 데이터 형식

### 초기 데이터 입력 예시
```csv
Promoter,CDS,Temperature,pH,Expression_Level
P1,Gene_A,30,7.0,120
P2,Gene_B,32,7.5,85
P3,Gene_A,28,6.8,95
P1,Gene_C,35,7.2,140
P2,Gene_B,30,7.0,110
```

### 생성되는 매핑 파일 예시
```csv
Well,Sample_ID,Promoter,CDS,Temperature,pH
A01,Sample_001,P1,Gene_A,30.5,7.1
A02,Sample_002,P2,Gene_B,31.2,7.3
A03,Sample_003,P3,Gene_C,29.8,6.9
```

## 🔧 기술 스택

- **머신러닝**: scikit-learn (Random Forest, Gradient Boosting, Linear Regression)
- **베이지안 최적화**: bayesian-optimization 라이브러리
- **데이터 시각화**: Plotly (인터랙티브 차트 및 그래프)
- **웹 인터페이스**: Streamlit (반응형 웹 UI)
- **데이터 처리**: pandas, numpy
- **모델 저장**: joblib (모델 직렬화)

## 📝 사용 시나리오 예시

### 시나리오 1: 새로운 단백질 발현 조건 최적화
1. **프로젝트 생성**: "Protein_X_Optimization" 프로젝트 생성
2. **초기 데이터**: 기존 20개 실험 결과 업로드
3. **모델 학습**: Random Forest가 최적 모델로 선택됨 (R² = 0.82)
4. **1차 사이클**: 베이지안 최적화로 8개 새로운 조건 제안
5. **실험 수행**: 매핑 파일로 실험 진행, 결과 입력
6. **모델 개선**: 새 데이터로 재학습 (R² = 0.89)
7. **추가 사이클**: 목표 발현량 달성까지 반복

### 시나리오 2: 기존 프로토콜 개선
1. **기존 데이터 활용**: 50개 기존 실험 결과로 시작
2. **베이스라인 설정**: 현재 최고 성능 조건 확인
3. **탐색적 실험**: 불확실성 높은 영역 우선 탐색
4. **점진적 개선**: 각 사이클마다 5-10% 성능 향상
5. **최적화 완료**: 더 이상 개선이 없을 때 종료
