# Protogen

Protogen은 Opentrons OT-2 및 Janus 로봇 시스템을 위한 액체 취급 프로토콜 자동화 Streamlit 애플리케이션입니다. DNA 조립을 위한 프로토콜을 생성하고, 사용자가 구성 요소, 볼륨 및 웰 위치를 정의할 수 있도록 지원합니다.

## 주요 기능

- **OT-2 및 Janus 프로토콜 자동 생성**
- **Excel 기반 구성 요소 가져오기**
- **사용자 정의 가능한 실험 기기 및 액체 처리 설정**
- **필요한 볼륨에 대한 실시간 피드백**
- **그룹 기반 조립 관리**

## 설치 방법

Python이 설치되어 있는지 확인하세요 (권장 버전: Python 3.8 이상). 그 후 다음 명령어로 필요한 라이브러리를 설치하세요:

```sh
pip install streamlit pandas opentrons openpyxl
```

## 사용 방법

다음 명령어를 실행하여 애플리케이션을 실행하세요:

```sh
streamlit run protogen.py
```

## UI 개요 및 입력 데이터 형식

### 1. 파일 업로드 (Input 파일 형식)
Protogen은 Excel 파일(`.xlsx`)을 입력으로 사용합니다. 각 시트(sheet)에는 96-well 플레이트 형식의 테이블이 포함되어 있으며, 각 셀에 해당 위치의 구성 요소 이름이 기록되어 있어야 합니다. 예를 들어:

|    | A  | B  | C  | D  |
|----|----|----|----|----|
| 1  | P1 | C1 | T1 |    |
| 2  | P2 | C2 | T2 |    |
| 3  | P3 | C3 | T3 |    |

- A1, B1, C1 등 각 웰에는 해당 구성 요소(Promoter, CDS, Terminator 등)의 이름이 입력됩니다.
- 빈 칸은 사용되지 않는 웰을 의미합니다.
- 자동으로 플레이트 이름과 구성 요소가 매핑됩니다.

### 2. 플레이트 및 볼륨 설정
- 업로드한 파일에서 플레이트 이름을 설정합니다.
- 각 웰의 기본 볼륨을 설정할 수 있습니다.
- 각 구성 요소의 초기 볼륨을 자동으로 할당합니다.

### 3. Lv1 조립 디자인
#### 입력 사항
- **그룹 설정**: TU(Transcription Unit)를 포함할 그룹을 정의합니다.
- **TU 디자인**: Promoter, CDS, Terminator, Connector를 선택합니다.
- **각 구성 요소의 볼륨 설정**: TU당 각 구성 요소의 사용량을 정의합니다.
- **공통 시약 추가**: GGAmix와 같은 공통 시약을 추가할 수 있습니다.

#### 계산 방식
- 업로드된 플레이트에서 필요한 구성 요소를 검색하여 조립할 조합을 생성합니다.
- 각 구성 요소의 사용량을 계산하여 최소 제작 횟수를 결정합니다.
- 볼륨 초과 시, 50µL 단위를 기준으로 반복 배치됩니다.

### 4. Lv2 조립 디자인
#### 입력 사항
- **Lv1에서 생성된 TU 리스트**
- **Lv2에서 필요한 볼륨 설정**: 각 TU당 볼륨 및 데드볼륨(dead volume)을 설정합니다.
- **Lv2 공통 시약 추가**: 백터(Vector)와 같은 추가 시약 설정이 가능합니다.

#### 계산 방식
- Lv1에서 생성된 TU의 사용량을 추적하고, 그룹별로 겹치지 않는 조합을 생성합니다.
- 각 TU의 사용 횟수를 계산하여 필요한 볼륨을 예측하고, 실험 수행을 위한 총 제작 수량을 결정합니다.

### 5. 프로토콜 생성 및 출력 파일
- **OT-2 또는 Janus 선택**: 실행할 로봇 시스템을 선택합니다.
- **프로토콜 자동 생성**: 선택한 설정을 기반으로 OT-2 또는 Janus 프로토콜을 생성합니다.
- **출력 확인**: 최종 출력 웰 위치 및 사용된 시약 정보를 확인합니다.

## 출력 파일

- **OT-2 프로토콜**: Opentrons에서 실행할 수 있도록 포맷된 Python 스크립트
- **Janus 프로토콜**: Hamilton Janus에서 사용할 수 있는 CSV 파일
- **업데이트된 재고 정보**: 업데이트된 볼륨 계산을 포함하는 데이터프레임

## 사용 예제

1. `plate_sample.xlsx` 파일을 업로드합니다.
2. 웰 위치 및 기본 볼륨 값을 설정합니다.
3. TU 및 그룹 설정을 정의합니다.
4. 볼륨 계산을 확인합니다.
5. OT-2 또는 Janus를 선택하고 프로토콜을 생성합니다.
6. 생성된 프로토콜을 저장하고 로봇 시스템에서 실행합니다.

## 문제 해결

- 프로그램이 부족한 볼륨을 경고하면 Excel 파일에서 재고 볼륨을 확인하고 업데이트하세요.
- OT-2 스크립트가 실행되지 않으면 API 호환성을 확인하세요 (예: `apiLevel: 2.17`).
- OT-2 설정에서 올바른 플레이트 정의가 적용되었는지 확인하세요.
